Aspects of Well Written Hooks

  • Clean API surface
  • Few (None) inputs
  • Does one thing well
  • Composes other hooks as building blocks
  • Doesn't leak implementation details

If something looks like a common pattern/problem
someone probably already wrote a hook for it

"There's an app Hook for that!"

Patterns to avoid

  • Many permutations of calling/being called
  • Managing too much state manually
  • More than one callback function
  • Two way hook communication/cycles (not a DAG)
